Создай полный рабочий проект Telegram WebApp игры с системой пользователей, ставками, балансами, реальными пополнениями и выводами. Используй:

- KV-хранилище для данных
- TON для реального пополнения и вывода (через tonapi.io)
- Telegram Stars для пополнения (через встроенную систему Stars, без payment provider)
- Админ‑панель с графиками, статистикой и управлением пользователями
- Обновление данных в реальном времени
- Красивый, простой и анимированный UI

====================================================
1. ХРАНЕНИЕ ДАННЫХ (KV)
====================================================
Используй KV-хранилище.

Пользователь — ключ user:<id>:
{
  "id": number,
  "first_name": string,
  "username": string,
  "joined_at": string,
  "last_seen": string,
  "balance": number,
  "banned": boolean,
  "stats": {
    "bets": number,
    "wins": number,
    "losses": number,
    "total_bet_amount": number,
    "total_win_amount": number
  },
  "activity": [
    { "time": string, "type": string, "value": number }
  ]
}

Ставка — bet:<id>  
Раунд — round:<id>  

Депозит TON — deposit_ton:<id>:
{
  "id": string,
  "userId": number,
  "amount": number,
  "txHash": string,
  "time": string
}

Депозит Stars — deposit_stars:<id>:
{
  "id": string,
  "userId": number,
  "amount": number,
  "invoice_payload": string,
  "time": string
}

Вывод TON — withdraw_ton:<id>:
{
  "id": string,
  "userId": number,
  "amount": number,
  "address": string,
  "status": "pending" | "completed" | "rejected",
  "time": string
}

====================================================
2. БАЗОВЫЕ ФУНКЦИИ ПОЛЬЗОВАТЕЛЯ
====================================================

Функция user-init:
- вызывается при запуске WebApp
- принимает данные Telegram пользователя
- если пользователя нет — создаёт user:<id>
- если есть — обновляет last_seen
- добавляет activity: { time: now, type: "visit" }

Функция get-user:
- возвращает профиль пользователя + баланс + stats

Функция update-user-balance(userId, delta):
- изменяет баланс
- обновляет stats при необходимости

====================================================
3. СТАВКИ И РАУНДЫ
====================================================

Функция place-bet:
- принимает userId, amount, roundId
- проверяет баланс
- уменьшает баланс
- создаёт bet:<id> с полями:
  { id, userId, roundId, amount, time, status: "pending" }
- добавляет activity: { type: "bet", value: amount }

Функция finish-round:
- принимает roundId, коэффициент, список победителей
- обновляет ставки:
  - выигравшим: статус "win", начисляет выигрыш
  - остальным: статус "lose"
- обновляет stats пользователя:
  - bets, wins, losses, total_bet_amount, total_win_amount
- создаёт round:<id> с результатами

====================================================
4. ПОПОЛНЕНИЕ ЧЕРЕЗ TON (tonapi.io)
====================================================

Используй tonapi.io.

Функция ton-create-deposit:
- принимает userId и сумму
- возвращает TON‑адрес для пополнения (можно использовать один общий адрес и метку/комментарий, либо отдельный адрес на пользователя — опиши реализацию)
- сохраняет ожидаемый депозит в KV (deposit_ton:<id> со статусом "pending")

Функция ton-check-deposits (периодический job):
- каждые N секунд опрашивает tonapi.io:
  GET https://tonapi.io/v2/blockchain/transactions?account=<address>
- ищет новые входящие транзакции
- если транзакция соответствует ожидаемому депозиту:
  - помечает deposit_ton:<id> как "completed"
  - пополняет баланс пользователя
  - добавляет activity: { type: "deposit_ton", value: amount }

UI:
- экран "Пополнить TON"
- поле суммы
- кнопка "Сгенерировать адрес"
- показ адреса + QR
- индикатор ожидания
- уведомление об успешном пополнении

====================================================
5. ВЫВОД ЧЕРЕЗ TON
====================================================

Функция ton-withdraw-request:
- принимает userId, amount, tonAddress
- проверяет баланс
- уменьшает баланс
- создаёт withdraw_ton:<id> со статусом "pending"
- добавляет activity: { type: "withdraw_ton_request", value: amount }

Функция ton-withdraw-approve (только для админа):
- принимает id заявки
- отправляет транзакцию через tonapi.io (опиши вызов sendTransaction / подход)
- обновляет статус на "completed"
- добавляет activity: { type: "withdraw_ton", value: amount }

Функция ton-withdraw-reject:
- возвращает сумму на баланс
- меняет статус на "rejected"

UI:
- экран "Вывод TON"
- поле суммы
- поле TON‑адреса
- подтверждение
- уведомление об отправке заявки

====================================================
6. ПОПОЛНЕНИЕ ЧЕРЕЗ TELEGRAM STARS
====================================================

Используй Telegram Stars без payment provider, через sendInvoice.

Функция stars-create-invoice:
- принимает userId и сумму (в Stars)
- создаёт invoice через Telegram Bot API sendInvoice:
  - provider_token: "STARS"
  - currency: "XTR"
  - prices: [{ label: "Пополнение", amount: amount * 100 }]
  - invoice_payload: "stars_deposit_" + userId + "_" + timestamp
- сохраняет запись deposit_stars:<id> со статусом "pending"
- возвращает invoice_id / ссылку / данные для WebApp

Webhook обработчик successful_payment:
- получает msg.successful_payment
- достаёт invoice_payload и total_amount
- находит соответствующий deposit_stars:<id>
- помечает как "completed"
- пополняет баланс пользователя на amount (total_amount / 100)
- добавляет activity: { type: "deposit_stars", value: amount }

UI:
- экран "Пополнить Stars"
- кнопка "Пополнить Stars"
- вызов Telegram.WebApp.openInvoice(invoice_id)
- уведомление об успешной оплате
